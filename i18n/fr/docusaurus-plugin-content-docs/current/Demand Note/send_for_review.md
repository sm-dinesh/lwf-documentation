# Envoyer la Note de demande pour révision

La fonctionnalité **Envoyer pour révision** permet aux utilisateurs de soumettre une Note de demande dans le flux de travail d'approbation. La note de demande sera examinée et approuvée par des utilisateurs désignés occupant différents rôles.

## Accéder à l'envoi pour révision

- Accédez à la page de modification de la Note de demande
- Cliquez sur le bouton **Envoyer pour révision** en haut de la page

## Aperçu

![Envoyer pour révision](/img/demand_note/demand_note_send_for_review.png)

Une fenêtre contextuelle apparaîtra dans laquelle les utilisateurs pourront sélectionner les approbateurs de différents rôles.

## Rôles d'approbation

La fenêtre modale contient trois niveaux d'approbation, indiqués dans l'image :

### 1. Réviseur
- Premier niveau de révision
- Responsable de la vérification initiale de la note de demande

### 2. Vérificateur
- Deuxième niveau de validation
- Assure l'exactitude et l'exhaustivité des données

### 3. Approbateur
- Autorité finale pour l'approbation
- Confirme et approuve la note de demande pour traitement ultérieur

## Envoi d'invitations

- Sélectionnez les utilisateurs pour chaque rôle (Réviseur, Vérificateur, Approbateur)
- Une fois sélectionnés, soumettez le formulaire
- Des invitations seront envoyées aux utilisateurs respectifs pour approbation

## Remarques

- Assurez-vous que les bons utilisateurs sont sélectionnés pour chaque rôle avant l'envoi
- Le processus d'approbation suit la hiérarchie définie : Réviseur → Vérificateur → Approbateur
- Après la soumission, les autorisations de modification peuvent être restreintes en fonction des paramètres du flux de travail
