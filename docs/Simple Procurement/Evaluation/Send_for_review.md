## Send for Review

![Send for Review Button](/img/simple_procurement/evaluations/invitation_page_send_for_review_button.png)

On the **Evaluation Summary page**, there is a button labeled **Send For Review**.

Clicking this button opens a modal to assign evaluation members.

---

## Send for Review Modal

![Send for Review Modal](/img/simple_procurement/evaluations/send_for_review_model.png)

This modal includes the following fields:

### 1. Select Currency
- Choose the currency for evaluation (e.g., EUR).

### 2. Select Evaluation Member
- Select one or more evaluation team members.

### 3. Send Button
- Sends the evaluation request to selected members.

**Behavior:**
- Once submitted, an **email notification is sent** to the selected evaluation members.

---

## Update Evaluation Team

![Update Review Button](/img/simple_procurement/evaluations/update_review_button.png)

After the first submission, the **Send For Review** button changes to **Update Evaluation Team**.

---

## Update Evaluation Modal

![Update Review Modal](/img/simple_procurement/evaluations/update_review_model.png)

This modal allows modification of the evaluation team.

### 1. Select Evaluation Team
- Add new members or modify existing ones.

### 2. Assigned Team List
- Displays all currently assigned evaluation members.

### 3. Status
- Shows whether each member is:
  - **Approved**
  - **Pending**

### 4. Notify Action
- Resends email notifications to selected members.

### 5. Update Button
- Saves changes to the evaluation team.

---

## Email Notification

![Email Notification Inbox](/img/simple_procurement/evaluations/email_notification_page.png)

- Evaluation members receive an **email notification**.
- The email contains a **Confirm Award** button.

Clicking this button redirects the user to the approval page.

---

## Approval Page

![Approve and Reject Page](/img/simple_procurement/evaluations/approve_and_reject_page.png)

This page allows evaluation members to review and take action.

### 1. Basic Information Tab
- Displays procurement details.

### 2. Bid Price
- Shows the bidder’s submitted price.

### 3. Evaluation Summary
- Summary of evaluation results (covered separately).

### 4. Approve Button
- Used to approve the procurement.

### 5. Reject Button
- Used to reject the procurement.

---

## Evaluation Summary (Approval View)

![Evaluation Summary on Approval](/img/simple_procurement/evaluations/evaluation_summery_page_on_approval.png)

- Displays evaluation details during the approval process.

---

## Approve Action

![Approve Modal](/img/simple_procurement/evaluations/award_approval_approve_modal.png)

When clicking **Approve**, a modal appears:

### 1. Notes
- Optional notes while approving.

### 2. Approve Button
- Confirms approval of the procurement.

---

## Reject Action

![Reject Modal](/img/simple_procurement/evaluations/award_approval_reject_modal.png)

When clicking **Reject**, a modal appears:

### 1. Notes
- Optional notes explaining rejection.

### 2. Confirm Button
- Confirms rejection of the procurement.

---

## Summary

This workflow ensures:

- Proper **assignment of evaluation members**  
- Controlled **review and update process**  
- Transparent **approval/rejection tracking**  
- Automated **email notifications and actions**  

The entire process is designed to keep procurement evaluation **structured, traceable, and auditable**.
